## App 定位

Driver Pay Pro 是給多元計程車與職業駕駛使用的本機優先收入、工時、支出與營運分析 App。核心情境是駕駛在手機上快速記錄每日工作資料，並從日、週、月與平台角度判斷實拿收入、淨收入、時薪與目標進度。

App 目前是繁體中文單頁式 PWA，資料只儲存在使用者目前瀏覽器的 `localStorage`，沒有帳號、伺服器資料庫或跨裝置同步。

## 開發理念

- 手機優先，支援最小 320px 寬度、單手操作與快速掃讀。
- 首頁採駕駛儀表板，而不是層層展開的傳統資料表單。
- 重要資訊直接顯示；能原位編輯就不開第二個輸入畫面。
- 適合自動儲存的選項不增加「完成」或「確認」步驟。
- 避免水平捲動、過度留白、卡片套卡片與無資訊價值的零值清單。
- 計算與資料相容性優先於視覺重構；UI 修改不得破壞既有資料。
- 已確認區塊預設凍結，只修改使用者明確指定的範圍。
- 規則式分析必須能由現有資料重現，不假裝已接入真正 AI。

## 專案範圍

目前產品範圍包括：

- 每日平台收入、小費、支出、班別、天氣、里程、訂單與備註紀錄。
- 開始、暫停、繼續、收工與手動工時。
- 每日、每週、每月、平台收入與目標統計。
- 月份明細的查看、編輯與刪除。
- 平台與支出類別管理、平台實拿比例、每日／每月目標。
- Yoxi 單筆收入明細與累加。
- 規則式「AI 營運助理」。
- CSV 匯出與 PWA 離線 App Shell。

## 資料模型與資料安全

唯一核心 storage key 為：

```text
driverPayApp.v2
```

頂層狀態：

| 欄位 | 內容 |
| --- | --- |
| `platforms` | 平台名稱陣列，可由 Driver Space 新增或移除 |
| `expenses` | 支出類別陣列，可新增或移除 |
| `entries` | 每日營運紀錄陣列 |
| `settings` | 每日／每月目標、平台實拿比例及相容性設定 |

每日 `entry` 目前可能包含：

- `id`、`date`、`shift`、`weather`、`note`
- `startTime`、`endTime`、`breakMinutes`、`manualHours`
- `tips`、`orders`、`km`
- `incomes`：以平台名稱為 key 的原始收入
- `expenses`：以支出類別為 key 的金額
- `workSession`：計時狀態、分段開始時間、累積工作／休息毫秒數與結束時間
- `incomeRecords`：Yoxi 單筆收入紀錄
- `weatherHistory`：為舊資料相容保留；目前主要顯示使用 `weather`

相容性規則：

- 不得更名或清除 `driverPayApp.v2`。
- 新欄位必須允許舊紀錄缺少該欄位。
- 讀取失敗時程式不會主動覆寫原始 localStorage。
- 天氣仍儲存文字值；月份明細 SVG 只屬顯示層。
- 不同瀏覽器、網域與裝置之間不會自動同步。

## 計算規則

### 收入

- 平台原始收入：`entry.incomes[平台]`。
- 平台實拿：`原始收入 × 平台實拿比例 ÷ 100`。
- 當日總收入：所有平台實拿合計加上小費。
- 當日淨收入：當日總收入減去全部支出。
- 平台分析與 AI 分析都使用實拿比例後的收入。

### 工時

計算優先順序：

1. `manualHours > 0` 時使用手動工時。
2. 有 `workSession` 時使用累積有效工作毫秒數；running 狀態加上目前分段。
3. 否則由開始／結束時間扣除休息分鐘；跨午夜時自動加 24 小時。
4. 資料不足時為 0。

平均時薪為 `淨收入 ÷ 工時`；工時為 0 時顯示 0 或「資料不足」，不得產生 NaN／Infinity。

### 目標

- 首頁進度：`今日總收入 ÷ 每日目標 × 100%`，百分比四捨五入為整數。
- 首頁進度條視覺最多填滿 100%，超標文字仍顯示真實百分比。
- 每日目標為 0、負數或異常值時顯示「尚未設定今日目標」。
- 月報目標以本月淨收入比較每月淨收入目標。

## 功能與頁面架構

底部主導覽目前有五個入口：

1. `今天`：Home 與每日輸入。
2. `月曆`：週報，並可切換月報與平台報表；同頁下方顯示月份明細。
3. `分析`：直接進入平台分析，也可切換週報與月報。
4. `AI`：規則式 AI 營運助理。
5. `Driver`：平台、支出、比例、目標與匯出設定。

## Dashboard

目前沒有獨立名為 Dashboard 的導覽頁。Dashboard 概念分布在：

- Home 深綠核心卡：今日收入、實際工時、目前時薪與每日目標。
- 工作狀態卡：總經過、休息、實際工時與計時控制。
- 週報／月報／平台分析：歷史績效儀表。
- AI 營運助理：規則式洞察與提醒。

舊版 hero 與「業績總覽」DOM 仍存在，但目前有效 CSS 將它們隱藏，不應視為使用者可操作功能。

## Home

首頁目前順序與行為：

1. 深綠核心卡。
2. 今日工作狀態與計時控制。
3. 平台收入直式清單。
4. 更新今日紀錄：日期、工時、支出與其他資料。
5. Yoxi 今日明細。

核心卡保留今日收入、今日實際工時、目前時薪；底部為右對齊目標文字與 8px 進度條。狀態文案支援未達標、剛好達標與超標。

班別與天氣位於「其他資料」內，使用直接選取按鈕並立即保存。日期列使用原生 date input。

## Calendar 與報表

月曆入口預設顯示週報：

- 月份篩選器。
- 以週一至週日分組。
- 週期、總額、淨收入、工時與平均時薪。
- 週期同年不重複年份，跨年才在兩端顯示年份。

報表切換器另提供：

- 月報：每月淨收入目標進度與每日收入相對走勢。
- 平台：本月平台實拿收入、占比與日均。

## 月份明細

月份明細保留月份下拉選單與同一份資料來源：

- 700px 以下顯示單一清單，不把每天做成獨立卡片。
- 第一行顯示日期／星期、班別、天氣 SVG 與當日總收入。
- 第二行只列收入大於 0 的平台，並保留編輯／刪除。
- 無平台收入時顯示「尚無收入紀錄」，當日總額仍顯示 `$0`。
- 每日紀錄以細水平線分隔；平台項目可自然換行。
- 700px 以上顯示桌面表格，包含平台、Tips、合計、淨收入與總計列。

月份明細天氣對應只影響手機清單：晴天、陰天、雨天、大雨與相容的多雲／雷雨／霧天文字會映射到 inline SVG；無資料或未對應值不顯示圖示。

## AI 功能

目前 AI 分頁是完全在瀏覽器執行的規則式「AI 營運助理」，沒有外部模型、聊天輸入、API Key 或 API 成本。

四個區塊：

- 今日建議：時薪相對本月平均、目標進度、主要平台、長工時與缺漏資料，最多顯示四項。
- 本月洞察：總收入、平均時薪、最佳收入日、最佳班別、最高收入平台與每月目標差距。
- 平台表現：平台本月實拿收入、占比、主要平台與近期占比增加摘要。
- 智慧提醒：低時薪、單日／連續長工時、里程或燃料費缺漏、三筆收入連續下降、平台占比增加及未達目標。

主要門檻：

- 今日工時達 12 小時：長工時建議。
- 今日時薪高於／低於本月平均 10%：顯示相對表現。
- 某日時薪低於本月平均 70%：異常提醒。
- 連續兩天各達 10 小時：連續長工時提醒。
- 最近三筆收入嚴格遞減：下降趨勢提醒。
- 至少四筆資料，近期平台占比增加 10 個百分點以上：平台趨勢提醒。

## 收入管理

- 所有平台同時顯示，一個平台一行。
- 一般平台輸入的是當日總額，新值覆蓋舊值。
- Yoxi 右側加號輸入單筆金額並累加；總額欄仍可直接修正。
- Enter 或失焦提交共用防重複機制。
- 輸入僅接受非負整數；非法數值不寫入。
- 原位儲存後立即更新首頁、報表、目標與 AI 分析。
- 小費計入總收入；支出只在淨收入中扣除。
- CSV 包含平台原始值、實拿值、比例與其他每日欄位。

## 平台管理

預設平台：Uber、LINE GO、Yoxi、55688、大都會、台灣大車隊、路邊客、自訂。

- 可新增不重複的平台名稱。
- 可移除平台，並刪除該平台目前的比例設定。
- 可設定 0–100% 的實拿比例；預設 55688 為 85%，其他預設 100%。
- 首頁固定優先採預設平台順序，自訂平台排在後方。
- 移除平台不會主動清除舊 entry 內的歷史鍵值，但目前畫面與計算只遍歷 `state.platforms`。

## 設定與 Driver Space

目前可用設定：

- 新增／移除收入平台。
- 新增／移除支出類別。
- 儲存平台實拿比例。
- 儲存每月淨收入目標與每日收入目標。
- 匯出 CSV。

頁面文字明確標示 Driver Space 詳細內容仍待後續開發；目前沒有帳號、個人檔案、車輛檔案或同步設定。

## PWA 與離線行為

- Manifest 使用 standalone、直向優先與繁體中文。
- Service Worker cache 名稱為 `driver-pay-pro-v04-official-master-icon-v1`。
- App Shell 包含首頁、manifest、主要 icon 與 hero 圖。
- fetch 採 network-first：成功時更新快取，失敗時讀快取，最後 fallback 到 `index.html`。
- activate 會刪除非目前名稱的舊快取並 claim clients。

## 非功能需求

- 手機最小寬度 320px，頁面不得水平 overflow。
- 390px 為主要手機驗證尺寸；700px 是月份明細手機／桌面分界。
- 固定底部導覽必須保留 safe-area 空間，不遮住最後內容。
- 表單控制至少 16px 字級，避免 iOS 自動放大。
- 金額採新台幣與千分位；資料不足不得顯示 NaN／Infinity。
- 所有動態文字輸出需經 escaping；互動控制需有可理解的 label。
- PWA 更新需驗證 Service Worker 不再提供舊版本。

## 目前不在產品中的能力

- 真正的生成式 AI、聊天、外部 AI API。
- 使用者帳號、登入、權限角色。
- 雲端資料庫、跨裝置同步、多人協作。
- CSV 匯入或自動備份還原。
- 原生 iOS／Android App。
- 自動化測試、TypeScript、ESLint、npm build。

## 驗收方式

現階段標準驗證方式：

1. 抽取 inline JavaScript 並執行語法檢查。
2. 執行 `git diff --check`。
3. 在實際瀏覽器操作收入、計時、編輯、刪除、設定與報表。
4. 檢查 Console error／warning。
5. 驗證 320px、390px 與至少 1024px。
6. 驗證重新整理後 `localStorage` 資料仍存在。
7. 針對 PWA 另檢查 manifest、Service Worker、離線與快取更新。

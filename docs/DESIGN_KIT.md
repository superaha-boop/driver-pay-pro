# Driver Pay Pro Design Kit

Version: 1.0

更新日期：2026-07-21

## 文件定位

本文件定義 Driver Pay Pro 的產品級 UI、UX、互動與品牌規則。現行尺寸與實作細節可在 `docs/UI_GUIDELINES.md` 查閱；若兩者與目前畫面不一致，先核對實際程式與核准 PRD，不得自行改版。

## Design Principles

### Driver First

設計必須符合真實司機工作流程。

### Mobile First

先確保 iPhone 直向畫面，再處理桌面。

### Speed First

減少點擊、等待與不必要步驟。

### Choose, Don't Type

能選擇就不要要求輸入。

建議元件：

- 時間：iOS Picker
- 日期：Date Picker
- 開關：Toggle
- 選項：直接選擇清單；需要容納更多內容時才考慮 Bottom Sheet
- 收入：數字鍵盤

### One-Hand First

主要操作必須容易單手完成。

### Apple Native First

優先採用 iPhone 使用者熟悉的操作模式，但不得複製 Apple 品牌素材。

### Today First

首頁優先呈現今天的狀況。

### Frequency First

使用越頻繁的功能，越靠近首頁。

### Quick Read

重要數字與平台名稱必須一眼可辨識。

### Every Tap Has Value

每一次點擊都必須有明確用途。

### Driver Safety

減少文字、誤觸與注意力負擔；不得鼓勵駕駛在車輛行進中操作。

## Mobile UI Rules

- 不允許水平捲動。
- 不顯示桌機側邊欄。
- 不要求手動縮放。
- 輸入框字體至少 16px。
- 重要按鈕需有足夠觸控面積。
- 重要數字使用較大字體。
- 避免過多小字。
- 避免狹窄多欄。
- 支援 iPhone Safe Area。
- 支援 PWA standalone。
- 高頻操作優先放在第一屏。
- 不可為塞入內容而全面縮小文字與按鈕。
- 主要驗收寬度為 320px、390px 與桌面 1024px。
- 固定底部導覽不得遮住內容或忽略 `safe-area-inset-bottom`。

## Visual System

### Color

| 用途 | 色彩 |
| --- | --- |
| App 背景 | `#F3F5F4` |
| 卡片 | `#FFFFFF` |
| 次背景 | `#F0F4F2` |
| 主文字 | `#17211D` |
| 次文字 | `#68736E` |
| 邊線 | `#DDE4E0` |
| 品牌綠 | `#15966F` |
| 深品牌綠 | `#087653` |
| 淺品牌綠 | `#E3F5ED` |
| 危險操作 | `#C8423B` |

- 收入資料使用深綠黑，不使用亮藍或各平台品牌色。
- 首頁主卡使用純色深綠，不加動畫或漸層。
- 危險色只用於刪除與錯誤。
- 色彩不能是唯一狀態訊號，需搭配文字。

### Typography

字體堆疊：

```css
-apple-system, BlinkMacSystemFont, "Segoe UI", "Noto Sans TC", sans-serif
```

- 首頁今日收入使用最大層級，現行為 `clamp(50px, 14vw, 64px)`。
- 一般頁標題約 24px；AI 頁標題 26px。
- 平台名稱約 24px、平台收入約 28px。
- 卡片標題 17–18px，主要內文 14–16px，輔助文字 12–14px。
- 金額與計時使用 tabular numerals。
- 主要金額不得被截斷；必要時允許換行而非縮成難讀小字。

### Card Style

- 一般卡片為白底、1px 淺灰綠邊框、18px 圓角與低強度陰影。
- 首頁核心卡為深品牌綠、白字、20px 圓角，現行高度約 214px，只允許因排版小幅調整。
- AI 今日建議卡為深綠主卡；其他 AI 區塊維持白卡。
- 避免卡片套卡片、每天一張獨立卡片或無資訊價值的過多留白。
- 月份明細每日 row 只用細線分隔，不加獨立背景、圓角或陰影。

### Icon

- 不使用 Emoji 代替功能或天氣圖示。
- 現行沒有安裝 icon 套件；需要 icon 時先沿用既有 SVG 或現有資產。
- Icon-only 按鈕必須有 `aria-label` 或等效無障礙名稱。
- 月份明細天氣使用約 18–20px、低彩度灰綠的 inline SVG；無資料時不顯示占位圖示。

### Progress Bar

- 首頁每日目標狀態文字置於進度條右上方並右對齊。
- 軌道高 8px、膠囊圓角，在深綠卡上使用足夠對比的半透明白色。
- 完成區接近不透明白色，視覺填滿限制 0–100%。
- 超標時文字可顯示真實百分比，進度區不得溢出。
- 無有效目標時顯示空軌道與「尚未設定今日目標」。
- 不使用動畫、漸層、粒子或額外圖示。

## Home Rules

首頁主要目的：

快速查看今日狀態及新增或修正平台收入。

資訊優先順序：

1. 今日收入
2. 今日實際工時
3. 目前時薪
4. 工作狀態
5. 平台快速輸入
6. 今日目標
7. Bottom Navigation

平台收入規則：

- 直接在首頁輸入。
- 不跳轉新頁。
- 不開啟多層 Dialog。
- 使用數字鍵盤。
- 一般平台輸入當日總額，新值覆蓋原總額。
- Yoxi 加號新增單筆收入並自動累加；點擊總額仍可直接修正累計值。
- 今日總收入與時薪同步更新。
- Yoxi 單筆新增完成後輸入框清空；一般平台原位顯示最新總額。
- 拒絕負數、NaN、Infinity 與非法金額；空白不應破壞既有資料。
- 錯誤收入必須能修改；Yoxi 單筆紀錄也必須能刪除。
- 不增加第二行 editor、鉛筆、逐列完成按鈕或平台切換卡。

目標：

一般使用情況下，新增一筆 Yoxi 收入或更新一般平台總額約 3 秒內完成。

## Work Time Rules

工作狀態：

- 尚未開始
- 開始跑車
- 暫停休息
- 繼續跑車
- 收工

計算：

- 開始後計算總經過時間。
- 暫停期間計入休息時間。
- 暫停期間不計入實際工時。
- 繼續後恢復實際工時計算。
- 收工後停止計時。
- 時薪＝淨收入 ÷ 實際工時。
- 工時為 0 時不得產生 NaN 或 Infinity。
- 重新整理或重開 App 不應失去當日狀態。

## Calendar and Detail Rules

- 手機月份明細第一行顯示日期／星期、班別、天氣 icon與當日總收入。
- 第二行顯示收入大於 0 的平台及編輯／刪除操作。
- 沒有平台收入時顯示「尚無收入紀錄」，當日總額 `$0` 仍保留。
- 每日之間使用 1px 細水平分隔線。
- 不使用每日獨立卡片、展開／收合或水平捲動。
- 700px 以下顯示手機清單；700px 以上保留桌面表格；不可同時顯示兩套內容。

## Responsive Rules

| Breakpoint | 規則 |
| --- | --- |
| 320–350px | 可縮小次要數字與控制，但仍保留可讀性和觸控面積 |
| 351–699px | 手機單欄、固定底部導覽、月份明細精簡清單 |
| 700px 以上 | 可使用雙欄與桌面月份表格 |

- 平台明細可自然換行，但平台名稱與其金額不可拆開。
- 兩套響應式呈現不可重複顯示相同資料。
- 桌面版正常不能取代 iPhone Safari／PWA 實機驗收。

## Brand Assets

Official Master Icon：

- 白色圓角方形背景
- 中央黑色圓形
- 白色可愛小車
- 綠色道路元素

正式主圖檔：

```text
assets/driver-pay-pro-master-icon.png
```

現行資產對應需注意：Header 使用 Master Icon；`manifest.webmanifest`、Apple Touch Icon 與 favicon 目前仍引用 `driver-pay-pro-icon-*` 系列。D-005 是統一品牌素材的已確認方向，但在另立 PRD 並完成 PWA 驗證前，不得順手改動資產路徑。

禁止：

- 重新生成 Logo
- 改變車子造型
- 改變道路
- 改變顏色
- 使用完整品牌展示板作為 Icon
- 任意裁切
- 用黑色填滿四角

所有 Logo 位置應使用現有正式品牌圖檔；任何品牌資產替換都必須先確認 manifest、Service Worker 快取、Apple Touch Icon 與 favicon。

## Accessibility

- 動態狀態不能只靠顏色表達。
- 進度條應保留 `role="progressbar"`、數值範圍與可理解的 `aria-valuetext`。
- 選擇按鈕的視覺狀態需同步 `aria-pressed`。
- 動態文字與平台名稱渲染前必須 escape。
- 保留 `prefers-reduced-motion` 支援。

## Frozen UI

除非核准 PRD 明確指定，不得順便修改：

- 首頁核心資訊結構與目標進度條位置
- 平台收入原位編輯及 Yoxi 加號累加流程
- 工作狀態與計時控制
- 班別與天氣直接選項
- 手機月份明細精簡 row
- 週報手機摘要第一輪響應式成果
- 規則式 AI 分頁四區塊順序

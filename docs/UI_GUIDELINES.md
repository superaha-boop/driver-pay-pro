## 設計方向

Driver Pay Pro 採穩重、清楚、低干擾的深綠與灰綠系統。主要使用情境是駕駛在手機上快速確認數字與完成輸入，因此資訊密度、點擊效率與戶外可讀性優先。

現行有效樣式以 `index.html` 後段的 `v0.4 Home — mobile-first driver console` 覆寫區為準；檔案前段仍存在部分舊版藍綠漸層規則，修改時不可誤把舊規則當成現行設計基準。

## 資訊與互動原則

- 首屏先呈現今日收入、工時、時薪與目標。
- 不為同一筆資料建立第二個編輯頁；收入採原位編輯。
- 直接選項優先於下拉選單，適用班別與天氣。
- 自動儲存優先於逐列完成按鈕。
- 只有次要詳細資料使用 `<details>` 收合；核心營運數據不收合。
- 空狀態使用簡短可行動文案，不堆出全部 `$0` 欄位。
- 手機不使用水平捲動解決主要內容排版。

## 色彩

### 現行核心 token

| Token／用途 | 色彩 |
| --- | --- |
| App 背景 `--bg` | `#F3F5F4` |
| 卡片 `--panel` | `#FFFFFF` |
| 次背景 `--panel-2` | `#F0F4F2` |
| 主文字 `--text` | `#17211D` |
| 次文字 `--muted` | `#68736E` |
| 邊線 `--line` | `#DDE4E0` |
| 品牌綠 `--brand` | `#15966F` |
| 深品牌綠 `--brand-strong` | `#087653` |
| 淺品牌綠 `--brand-soft` | `#E3F5ED` |
| 危險操作 `--danger` | `#C8423B` |
| 卡片陰影 `--shadow` | `0 4px 18px rgba(26, 48, 38, .06)` |

常用補充色：

- 深綠黑資料文字：`#203128`、`#17211B`。
- 低彩度灰綠：`#7E8982`、`#98A29C`。
- 清單分隔線：`#E3E9E5` 或 `#EDF1EF`。
- 輸入框背景／邊框：`#FAFCFB`／`#DCE4DF`。

規則：

- 收入資料不使用亮藍色或各平台品牌色。
- 首頁主卡使用純色深綠，不加入動畫或漸層。
- 危險色只用於刪除與錯誤，不用作一般強調。
- 對比不足的灰色不可承載主要金額。

## Typography

字體堆疊：

```css
-apple-system, BlinkMacSystemFont, "Segoe UI", "Noto Sans TC", sans-serif
```

現行主要層級：

| 元素 | 建議／現行尺寸 |
| --- | --- |
| AI 頁標題 | 26px |
| 一般頁標題 | 約 24px |
| 首頁今日收入 | `clamp(50px, 14vw, 64px)` |
| 首頁工時／時薪 | `clamp(26px, 7vw, 32px)`；極窄螢幕 22px |
| 平台名稱 | 24px、700 |
| 平台收入 | 28px、700；小螢幕 26px |
| 卡片標題 | 17–18px |
| 清單日期／總額 | 18px、700 |
| 主要內文 | 14–16px |
| 輔助文字／標籤 | 12–14px |

金額與計時數字使用 tabular numerals；大數字允許 `overflow-wrap`，不可截掉金額。

## Icon

目前沒有安裝 icon 套件。

- App icon 使用 `assets/driver-pay-pro-*` 與 `driver-pay-pro-master-icon.png`。
- 月份明細天氣使用程式內 inline SVG，不使用 Emoji。
- 天氣 SVG 約 19px、`currentColor`、1.8px stroke，顏色為低彩度灰綠。
- 對應：Sun、CloudSun、Cloud、CloudRain、CloudLightning、CloudFog。
- 無天氣資料或未對應文字不顯示占位 icon。
- SVG 必須保留 `role="img"`、`aria-label` 與 `<title>`。
- 底部導覽目前使用文字符號作為 icon；TODO：是否統一改為同一套 SVG icon 尚未決定。

## Card Style

### 一般卡片

- 白底、`1px solid var(--line)`。
- 手機主要圓角 18px。
- 陰影使用 `var(--shadow)`，保持低強度。
- 卡片間距通常 8–12px；避免卡片套卡片。

### 首頁核心卡

- 深綠 `--brand-strong`、白字、20px 圓角。
- 約 214px 高；只允許為排版做小幅調整。
- 無陰影，讓核心卡靠色塊建立層級。

### AI 今日建議卡

- 深綠底、白字、18px 圓角、無陰影。
- 其他 AI 卡為白底、細邊框與輕陰影。

### 月份明細

- 整個月份明細由一張外層面板承載。
- 每日 row 不加圓角、背景或獨立陰影，只用細線分隔。

## 按鈕與輸入控制

- 表單字級至少 16px，避免 iOS focus zoom。
- 一般按鈕最小高度約 40px；主要儲存按鈕 52px。
- 平台收入框高 56px、15px 圓角。
- Yoxi 加號 44×44px；320–350px 可降為 42×42px。
- focus 使用深綠邊框與淡綠背景；不要改成亮藍色。
- 刪除前保留確認對話框。
- 收入列表不加入鉛筆、逐列完成按鈕或第二行 editor。

## Progress Bar

### 首頁每日目標

- 狀態文字置於進度條右上方並右對齊。
- 軌道 8px、膠囊圓角，白色 24% 透明度。
- 完成區為接近白色的 96% 不透明度。
- 進度條接近卡片內容完整寬度。
- 視覺填滿限制 0–100%；文字可顯示真實超標百分比。
- 無目標時顯示空軌道及「尚未設定今日目標」。
- 不使用動畫、漸層、粒子或額外 icon。

### 月報與平台報表

- 一般軌道基準 12px；620px 以下的既有規則為 14px。
- 現有報表完成區使用品牌綠、藍與黃的歷史漸層；若要統一成純綠，需另立需求，不應順手修改。

## Responsive 規範

### 全域

- HTML／body 最小寬度 320px，body 禁止水平 overflow。
- 主要內容最大寬度 760px，置中顯示。
- 固定底部導覽為五欄，需納入 `env(safe-area-inset-bottom)`。
- 主內容底部 padding 必須避免被導覽列遮住。

### Breakpoints

| Breakpoint | 現行用途 |
| --- | --- |
| `max-width: 350px` | 縮小首頁次要數字、平台欄寬與 Yoxi 按鈕 |
| `max-width: 620px` | 舊版手機表單與報表細節；仍受後段 v0.4 規則覆寫 |
| `max-width: 699px` | 月份明細顯示精簡每日清單，隱藏桌面表格 |
| `min-width: 700px` | AI 月摘要三欄、一般欄位雙欄、月份明細桌面表格 |
| `max-width: 980px` | 舊版側欄轉換規則；後段 v0.4 已把導覽固定為底部模式 |

必要測試寬度：320px、390px、1024px。320px 允許平台明細自然換行，但平台名稱與金額不可拆開。

## 月份明細規範

- 第一行：日期／星期、班別、天氣 icon；右側當日總收入。
- 第二行：只顯示收入大於 0 的平台；右側編輯與刪除。
- 無平台收入：顯示「尚無收入紀錄」。
- 保留 `$0` 總額與維護操作。
- 每筆間使用 `#E3E9E5` 1px 分隔線，最後一筆無線。
- 不加入展開、收合、查看更多、水平捲動或每日獨立卡片。
- 700px 以上保留桌面表格，兩套顯示不可同時出現。

## AI 分頁規範

- 順序固定為：今日建議、本月洞察、平台表現、智慧提醒。
- 今日建議是唯一深綠主卡；其餘維持白卡。
- 月摘要手機兩欄、700px 以上三欄。
- 平台列使用名稱、金額、占比三欄，過長名稱省略但金額不可換行。
- 不加入聊天框、「詢問 AI」按鈕或虛假的連線狀態。
- 空資料時直接顯示清楚的空狀態，不產生推測內容。

## 內容與格式

- 介面使用繁體中文。
- 金額使用 `zh-TW` 新台幣格式與千分位，最多 0 位小數。
- 工時首頁使用「X 小時 YY 分」；報表可用一位小數小時。
- 週期：同年 `7/12－7/17`，跨年 `2026/12/29－2027/1/4`。
- 資料不足顯示「資料不足」或具體空狀態，不用「未知」或問號 icon。
- 提醒文案應指出資料、方向與可採取行動，避免自稱模型推論。

## 無障礙

- icon-only 按鈕必須有 `title` 或 `aria-label`。
- 進度條使用 `role="progressbar"`、min／max／now 與 `aria-valuetext`。
- 選項群組使用可理解的 `aria-label`，選取狀態同步 `aria-pressed`。
- 動態產生內容必須 escape；屬性值使用 attribute escaping。
- 保留 `prefers-reduced-motion` 規則。
- 色彩不可是唯一狀態訊號；目標、工作狀態與提醒都需文字描述。

## 凍結區塊與禁止回退設計

- 首頁核心資訊結構、平台收入原位編輯與計時流程預設凍結。
- 一般平台維持總額覆蓋；Yoxi 維持單筆累加。
- 不恢復平台切換卡、第二行 editor、鉛筆、逐列完成按鈕或亮藍收入字。
- 不恢復首頁獨立天氣列或「先選天氣才能開始工作」。
- 不把月份明細退回超寬手機表格、每日大型卡片或全部零值平台。
- 視覺修改不得改變 `driverPayApp.v2` 或收入／工時／統計公式。
